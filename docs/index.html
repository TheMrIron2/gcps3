gcps3 is a runtime project for bringing GameCube-style source ports, samples, and systems to native platforms, with PlayStation 3 as the defining target.

Mission

Make GameCube software realistic to port, play and enhance.

gcps3 exists to reduce the distance between GameCube decompilation projects and PlayStation 3. The project is not trying to emulate a console, but building the intermediary layer a source port needs in order to stand on its own.

The goal is to provide familiar GameCube-style APIs, map them to native services, validate behaviour on PC, and turn the PS3 into a serious target for 6th generation ports. These ports will have the opportunity to run at high resolutions and bring visual enhancements to these games.

Along the way, gcps3 should strengthen the PS3 homebrew ecosystem itself: clearer examples, better documentation, and reusable knowledge that serves and tests each area of tools. It's a thorough stress test of PSL1GHT and the toolchain.

Approach

Source-level portability, not emulation.

gcps3 follows a compatibility-layer model - keep the public surface familiar to GameCube software, then implement that surface through native layers.

Systems

The runtime layer a port can build on.

gcps3 is organised around the platform systems that source ports need: startup, logging, time, files, input, graphics, and eventually audio and suitable PS3 acceleration paths.

Runtime core

A small foundation for startup, shutdown, configuration, logging, assertions, error reporting, and predictable sequencing.

Platform APIs

GameCube-style OS, PAD, and DVD/filesystem layers that let source code express intent without binding directly to one host platform.

GX frontend

A graphics frontend that captures GameCube-inspired rendering state while keeping renderer implementation details behind explicit backend boundaries.

PS3 backends

A path toward native PS3 support through PSL1GHT, controller APIs, filesystem access, RSX rendering, and SPE-friendly jobs.

Renderer direction

From a GX frontend to an RSX backend.

gcps3 treats graphics as a translation problem. The GX frontend should describe intent and state. Backends should decide how that state is interpreted.

gxnull provides a dependency-free correctness path. gxpc provides a fast visual feedback loop. gxrsx is the long-term PS3 renderer backend: the place where GX state becomes native RSX work.

The goal is not to copy a desktop renderer onto PS3. The goal is to respect the hardware, learn from examples, and shape a renderer tailored to the console.

Principles

The project should be ambitious, but not vague.

  • Prefer source-level portability over binary compatibility.
  • Keep public APIs small, understandable, and GameCube-oriented.
  • Keep frontend runtime code independent from PC and PS3 backend details.
  • Use PC validation to improve efficiency of PS3 development.
  • Learn from Aurora-style compatibility layers while keeping PS3 constraints central.
  • Choose correctness, debuggability, and clear logging before optimisation.
  • Use SPEs where the hardware makes sense: audio, ADPCM, texture conversion, swizzling, matrices, and animation batches.

Long-term vision

A practical bridge between decompilation work and PS3 homebrew.

A mature gcps3 should make GameCube source ports less dependent on particular platform work. It should offer a useful subset of GameCube runtime behaviour, reliable native PS3 paths, and clear documentation.

Selected ports

Make desired GameCube decompilation projects more realistic to adapt to PS3 without reinventing the platform layer each time.

Useful GX subset

Create a practical implementation of GX rendering that is suited to the PS3 hardware.

PS3-native knowledge

Form lessons about RSX, PSL1GHT, input, filesystem, audio, memory, and performance in reusable examples and documentation.

Future flexibility

Keep the runtime GameCube-first while avoiding assumptions that would make suitable Wii-style software impossible to explore later.

Contribute

Help create the next generation of ports.

gcps3 needs contributors who care about clean interfaces, graphics translation, PS3 hardware, decompilation projects, runtime design, documentation, and small examples that prove larger ideas.
